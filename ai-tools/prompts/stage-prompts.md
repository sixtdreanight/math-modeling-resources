# 分阶段提示词模板

## 1. 选题与问题分析

### 1.1 赛题理解
```
你是一名数学建模竞赛专家。我正在参加美赛/国赛，赛题如下：

【粘贴完整题目，包括附件描述、数据说明】

请帮我完成：
1. 用一句话概括每个问题的核心
2. 拆解为可独立建模的子问题
3. 指出每个子问题的关键约束和隐含假设
4. 推荐每个子问题的候选模型（至少2个）
5. 评估每个子问题的难度（★1-5）

输出格式：结构化的分析表格。
```

### 1.2 文献速览
```
请帮我快速了解以下研究领域的核心方法和最新进展：
【领域关键词】

要求：
- 列出 3-5 个最相关的经典模型
- 每个模型一句话说明核心思想
- 指出这些模型的关键假设和局限性
- 给出来源（期刊/会议名称），我会自行核实
```

### 1.3 选题决策矩阵
```
我们团队的情况：
- 数学基础：【强/中等/较弱】
- 编程能力：【强/中等/较弱】
- 英文写作：【强/中等/较弱】

6道题的粗略评估：
A题：【一句话描述 + 你的难度预估】
B题：【同上】
...

请帮我从以下维度给每道题打分（1-5）：
- 数学难度匹配度
- 编程实现难度
- 数据可得性
- 创新空间
- 总体推荐度
```

## 2. 模型构建

### 2.1 模型推选
```
题目核心需求：【简述】
可用数据类型：【描述】
预期输出：【描述】

请推荐 3-5 个候选模型，对每个模型列出：
- 核心公式（LaTeX 格式）
- 为什么适合本题
- 数据需求
- 计算复杂度
- 优缺点
- 推荐指数（★1-5）

最后给出你的首选推荐和理由。
```

### 2.2 模型改进（"拉踩法"）
```
基准模型是【模型名】，其已知缺陷包括：
1. ...
2. ...

针对每个缺陷，请提出一个具体的改进方案：
- 改进思路
- 数学公式（LaTeX）
- 预期改进效果
- 实现复杂度

同时给改进后的模型起一个学术化的名字（如 "自适应加权XXX模型"）。
```

### 2.3 组合创新公式
```
# 基准模型 + 增强模块 = 新模型

示例：
- 随机森林 + 注意力加权   = 注意力加权随机森林
- ARIMA    + GARCH残差修正 = ARIMA-GARCH混合预测
- TOPSIS   + 熵权自适应    = 自适应熵权TOPSIS

请按照这个模式，帮我设计 3 个针对本题的创新模型组合。
要求：每个组合要有学术合理性，能自圆其说。
```

### 2.4 假设与符号系统
```
基于我们选定的模型【模型名】，请：
1. 列出该模型需要的基本假设（至少 5 条）
2. 对每条假设给出合理性论证（1-2 句）
3. 建立完整的符号表（LaTeX 格式）

要求：假设不要过于理想化，保留一定现实约束。
```

## 3. 代码实现

### 3.1 算法实现
```
请用 Python 实现【算法名称】。

要求：
- 使用 numpy/scipy/pandas 等常用库
- 独立可运行的函数封装
- 写好输入输出的类型和注释
- 包含 3 个以上的测试用例
- 处理常见异常（除零、空值、维度不匹配等）
- 输出结果格式化为可直接用于论文的样式
```

### 3.2 代码调试
```
以下代码运行报错：
【粘贴代码和完整错误信息】

请：
1. 分析错误根因（不是表面现象）
2. 给出修正后的完整函数
3. 解释为什么这样修复可行
```

### 3.3 可视化生成
```
请用 matplotlib/seaborn 生成以下图表：

1. 折线图：实际值 vs 模型拟合值（含置信区间阴影）
2. 热力图：参数灵敏性分析结果矩阵
3. 雷达图：多方案对比（5 个指标）
4. 柱状图：方案排序结果

要求：
- 配色学术化（避免默认配色）
- 分辨率 ≥ 300 dpi
- 支持中文/英文切换
- 生成独立函数，传数据即可出图
```

## 4. 论文写作

### 4.1 摘要撰写
```
你是 MCM/ICM 评审专家。我们完成了以下工作：

问题：【1句】
方法：【2-3句，列出模型名称】
关键结果：【2-3句，含具体数字】
结论：【1句】

请撰写英文摘要，要求：
- 250-300词
- 不为超出实际工作范围的内容添油加醋
- 学术英文，无语法错误
- 采用 "we developed...", "our results show..." 的自然风格
- 避免 "Firstly, Secondly, Moreover, Notably, It is worth noting that" 
  等模板化表达
```

### 4.2 学术润色
```
请将以下段落润色为学术英文：

【原文】

要求：
- 保持原意，不添加新内容
- 使用正式但不生硬的学术表达
- 提升句子结构多样性
- 避免过度使用被动语态
- 使文本读起来像母语学者的自然写作
```

### 4.3 灵敏性分析模板
```
模型的关键参数有：【参数列表】

请帮我设计灵敏性分析框架：
1. 每个参数的合理变化范围（给出依据）
2. 采用什么灵敏性分析方法（OAT / Morris / Sobol / LHS，说明理由）
3. 用什么指标衡量灵敏性
4. 结果如何展示（表格/图表建议）
5. 一句话概括可写进论文的结论模板
```

### 4.4 局限性与未来工作
```
我们的模型存在以下局限：
1. ...
2. ...
3. ...

请用学术英文撰写模型的 Limitations and Future Work 部分，要求：
- 3-4 个局限性，每个都要诚实而不过分自贬
- 每个局限性给出 1 个可行的改进方向
- 不写"由于时间不够"（竞赛的万能借口评审看腻了）
- 150-200 词
```

## 5. 全流程系统提示词

### 5.1 全能解题模式
```
I am a contestant of MCM/ICM 2026. I need to solve:

【Paste complete problem text + all attachments + data descriptions】

My team's background: 
- Math level: 【basic/intermediate/advanced】
- Programming: 【basic/intermediate/advanced】
- Writing: 【basic/intermediate/advanced】

Please guide us through the complete workflow:
1. Problem decomposition → 2. Model selection → 3. Solution design → 4. Verification → 5. Writing

For each stage:
- Present options with trade-offs
- Flag common mistakes for our level
- Suggest specific models/formulas (LaTeX)
- Recommend code libraries
- Point out where sensitivity analysis is needed
```
