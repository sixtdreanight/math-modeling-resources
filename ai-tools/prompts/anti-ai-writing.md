# 去 AI 化写作提示词

## 为什么需要？

AI 生成的文本有独特"AI味"——结构雷同、连接词重复、缺少真实思考痕迹。美赛评审每天读几十篇论文，一眼能看出哪篇是 AI 写的。**让 AI 帮你写得像人**，不是让人帮你用 AI。

## 核心策略：角色设定

```
请以以下角色的口吻撰写：

"一位有 15 年经验的数学建模教授，习惯于在课堂上用具体例子解释抽象概念，
在论文中使用准确的限定词而非绝对断言，偶尔展现对模型局限性的诚实思考。"
```

## 风格控制提示词

### 词汇层面
```
1. "避免使用以下 AI 高频词：notably, moreover, furthermore, 
    it is worth noting that, in this regard, a plethora of, 
    delve into, underscore, showcase"
2. "将'utilize'替换为'use'"
3. "将'demonstrate'替换为'show'（除非是严格证明）"
4. "减少被动语态的使用比例（保持在 30% 以下）"
5. "每段至少使用一个具体的数值来描述结果"
```

### 句法层面
```
6. "每 3-4 句话变化一次句子长度，穿插短句制造节奏感"
7. "不要在段落开头连续使用连接词"
8. "首句不要用'In this section, we...'这种模板化开头"
9. "使用'we'但不要每句话都以'We'开头"
```

### 段落层面
```
10. "每段应以该段最关键的发现开头（而非罗列该段要做什么）"
11. "段落之间应有逻辑过渡，但不要用'As mentioned above'"
12. "技术段落中穿插一句直观解释，像教授在课堂上那样"
```

### 思维层面（最重要）
```
13. "展现出对模型局限性的诚实思考，而不是急于说模型有多好"
14. "在合适的地方加入'这可能是因为...'的推测性分析"
15. "引入适量的学术保留态度词汇：suggest, indicate, appear to
     而非 prove, demonstrate conclusively"
16. "结果分析中区分'统计显著'和'实际显著'"
17. "对意外结果给出可能的解释而非视而不见"
```

## 分阶段去 AI 化

### 摘要去 AI 化
```
请重写以下摘要。要求：
- 第一句话直接说出解决了什么问题（不要铺垫）
- 不出现"This paper presents", "In this paper, we"
- 用一句具体量化结论收尾
- 缩短句子平均长度
```

### Introduction 去 AI 化
```
请重写 Introduction。要求：
- 不要以笼统的"With the development of..."开头
- 研究缺口描述要具体（引用真实数据）
- 贡献陈述要量化（不要只说"improved"，要说"improved by X%"）
```

### 模型描述去 AI 化
```
请重写模型描述。要求：
- 公式前后加入直觉解释（"通俗地说，这个公式表达的是..."）
- 每个假设后面加一句"如果违反这个假设，会怎样"
- 减少堆砌符号，增加对符号含义的解释
```

### 结论去 AI 化
```
请重写结论。要求：
- 不要简单重复摘要和结果
- 加入对未来参赛者有用的反思
- 给出具体的改进方向而非"未来可以进一步研究"
```

## 万能去 AI 指令

把以下指令加在任何写作提示词末尾：

```
写作要求：
1. 避免使用 AI 写作的标志性表达（notably, moreover, a plethora of 等）
2. 句子长度要有变化（短句 5-10 词穿插长句 20-30 词）
3. 每段包含至少一个具体量化信息
4. 适当展现学术诚实性（承认局限、使用限定词）
5. 最终阅读体验应该像一位有经验的研究者写的文章，而非教科书或综述
```
